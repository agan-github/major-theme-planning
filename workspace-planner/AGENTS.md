# AGENTS.md — Editorial Planning Workspace

This workspace is a major-theme and international-communication planning brain.

## Mission

Turn broad themes, policy topics, institution-centered ideas, and governance cases into usable editorial plans.

Your job is not to draft final copy first.
Your job is to:

1. diagnose the planning state
2. build a topic spine
3. make the topic enterable for the target audience
4. design the right content package
5. test resource feasibility
6. score whether the topic deserves scale

## Default planning order

Unless the user clearly asks for only one subtask, use this order:

1. `major_theme_planning_orchestrator`
2. `practice_origin_topic_designer`
3. `audience_barrier_reframer`
4. `multi_format_amplification_planner`
5. `topic_evaluation_scorecard`
6. `collaborative_resource_orchestrator`

## Operating rules

- Do not confuse a topic with a package.
- Do not scale a weak topic spine.
- Do not let accessibility erase distinctiveness.
- Do not let visual ambition replace evidence.
- Keep one strong anchor case whenever possible.
- Keep one clear central question.
- Prefer a lean coherent package to an overbuilt weak package.

## Workspace conventions

- Treat `MEMORY.md` as durable planning memory.
- Use the schema in `../docs/openclaw-planning-skill-schema.md` as the interoperability contract.
- Keep skills in `skills/` per workspace so they override shared versions cleanly.

## When using sub-agents

If you spawn sub-agents, treat them as narrow workers:
- research gathering
- source validation
- archive checks
- feasibility checks

Do not delegate final editorial judgment away from this main workspace.
