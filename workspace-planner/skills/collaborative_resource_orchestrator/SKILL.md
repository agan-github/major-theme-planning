# Collaborative Resource Orchestrator

Turn a topic plan into a resource architecture.

## What this skill does

Use this skill when the main question is not only what the angle is, but whether the topic can actually be executed well.

## Working method

1. Restate the topic as an execution problem.
2. Separate resource categories.
3. Distinguish support resources from shape-determining resources.
4. Map the collaboration structure.
5. identify gaps and fallback shapes.
6. Return a resource orchestration memo.

## Output requirements

Always include:
- topic spine
- resource logic
- resource categories
- shape-determining resources
- collaboration map
- essential vs strengthening resources
- resource gaps
- minimum viable resource configuration
- topic-shape consequences
- coordination risks
- final resource verdict

## Rules

- follow topic logic, not generic inventory logic
- rank resources by importance
- separate proof-making resources from polishing resources
- explain how missing resources change the topic
- prefer smaller coherent plans to resource fantasy
