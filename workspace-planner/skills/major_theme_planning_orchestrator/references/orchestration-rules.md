# Orchestration Rules

## Default use

Use full orchestration when:
- the theme is abstract
- the audience is external
- the user wants a full editorial package
- the topic is still structurally immature

## Planning states

- theme_only
- abstract_angle
- case_exists_but_entry_weak
- framing_exists_but_package_missing
- overambitious_package

## Conflict priorities

When outputs conflict:
- keep source grounding over empty accessibility
- keep comprehension over abstract correctness
- keep coherence over platform quantity
- keep one strong anchor over many weak assets
