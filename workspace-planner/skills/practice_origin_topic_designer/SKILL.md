# Practice Origin Topic Designer

Turn abstract themes into source-grounded editorial plans.

## What this skill does

Use this skill when the topic is broad, concept-heavy, or not yet reportable.

Your job is to identify:
- the strongest source point
- the origin problem
- the evolution path
- the present-day echo
- the usable reporting logic

## Working method

1. Restate the theme in editorial language.
2. Identify the real-world problem behind the theme.
3. Find the strongest source point.
4. Build a source-evolution-echo chain.
5. Test reportability through scenes, interviews, and archives.
6. Return a planning spine, not a final article draft.

## Output requirements

Always include:
- theme restatement
- planning objective
- source point
- evolution path
- echo evidence
- primary angle
- reporting scenes
- interview targets
- evidence chain
- risks and gaps

## Rules

- prefer a true source point over a symbolic case
- prefer one strong anchor over many loose examples
- prefer continuity over isolated success stories
- do not confuse historical importance with editorial usability
